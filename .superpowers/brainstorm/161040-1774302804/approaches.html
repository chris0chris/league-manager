3 Design Approaches for Template Library

All replace the AppHeader "Generate Tournament" button with a "Templates 📚" entry point

A

Two-Panel Modal Recommended

Full-screen modal with a list/preview split. Left: searchable, filterable template list grouped by type. Right: live preview with action buttons. Tournament formats are a built-in group; saved server-side templates are separate groups.

Templates | | | ▶ Regional Cup | | +-------------------+-------------------------------------+

Pros

  • Single entry point for all schedule structures
  • Preview before applying
  • Full CRUD accessible in one place

Cons

  • More complex to build
  • Tournament config needs a sub-step
B

Wizard Stepper Modal

Step-by-step guided flow: choose type → browse & select → configure → apply. Good for onboarding but adds extra clicks for experienced users.

+--[Templates]------------------------------------[X]+ | [1. Type] → [2. Browse] → [3. Configure] → [4. Apply] | |------------------------------------------------------| | Step 1: What do you want to do? | | | | ┌──────────────────┐ ┌──────────────────┐ | | │ 🏆 Tournament │ │ 📁 Load saved │ | | │ format │ │ template │ | | └──────────────────┘ └──────────────────┘ | | | | [Next →] | +------------------------------------------------------+

Pros

  • Very clear flow for new users

Cons

  • Extra clicks for experienced users
  • Templates & tournament formats still feel separate
C

Toolbar Dropdown + Separate Manage Modal

Quick-access dropdown from toolbar for common actions (recents, save, generate), plus a separate full-screen "Manage Templates" modal for CRUD. Two tiers of access.

FlowToolbar: [Templates ▾] ├── Recent: 3v3 Format ├── Recent: Club Standard ├── ───────────────────── ├── 🏆 Generate Tournament... ├── 💾 Save as Template... └── 📁 Manage Templates → (full modal)

Pros

  • Fast access for power users
  • Recents save clicks

Cons

  • Two surfaces to build and maintain
  • More fragmented UX
