Detailed Design: Database Guard

1. DatabaseGuardMiddleware

  • Check: SELECT 1 via Django connection.
  • Caching: 10s TTL for success, 5s for failure to avoid hammering DB.
  • Action: Redirect to /database-error/ if down.

2. Health Check (200 OK Policy)

  • Logic: Catch DB exceptions in health check.
  • Response: 200 OK with {"status": "degraded"} body.
  • Benefit: Container stays alive; status is visible.

3. Static Info Page

  • URL: /database-error/
  • Constraint: Zero DB queries in template/view.
